# 前端说明

## 技术栈

前端位于：

- `aitrade-fe/`

当前技术栈：

- Vue 3
- TypeScript
- Vite
- Pinia
- Vue Router
- ant-design-vue

## 启动方式

```bash
pnpm --dir aitrade-fe dev
```

默认本地开发地址：

- `http://127.0.0.1:5173`

构建命令：

```bash
pnpm --dir aitrade-fe build
```

## 页面结构

当前管理台包含以下页面：

- 登录页：`/login`
- 交易日志：`/trade-logs`
- 策略配置：`/strategies`
- 历史数据管理：`/backtest-data`
- 策略回测：`/backtests`
- 系统设置：`/system-settings`
- 系统日志：`/system-logs`
- 用户维护：`/users`

默认登录后首页会跳转到：

- `/trade-logs`

## 路由与登录态

路由定义位于：

- `aitrade-fe/src/router/index.ts`

登录态由 Pinia 管理：

- `aitrade-fe/src/stores/auth.ts`

关键行为：

- 访问非登录页前会尝试恢复 token 对应的当前用户
- 未登录访问业务页时会跳转到 `/login`
- 已登录访问 `/login` 会重定向到 `/trade-logs`

## HTTP 与代理

Axios 封装位于：

- `aitrade-fe/src/api/http.ts`

开发环境中，`VITE_API_BASE_URL` 默认为空，前端统一请求相对路径 `/api/...`，再通过 Vite 代理转发到后端。

Vite 代理配置位于：

- `aitrade-fe/vite.config.ts`

当前会代理：

- `/api`
- `/health`

目标地址默认为：

- `http://127.0.0.1:18080`

## 页面能力

### 登录页

- 用户名、密码、图形验证码登录
- 前端包含字段级必填校验
- 图形验证码通过后端 `POST /api/auth/captcha` 获取

### 交易日志页

- 支持交易日志分页查询
- 当前筛选项为：策略、方向、结果、交易对、交易时间范围
- 筛选项尽量使用固定选项，减少手工输入
- 支持查看当前持仓抽屉
- 表格中的方向和结果已做标签化展示

### 策略配置页

- 展示策略定义和策略配置列表
- 支持创建、编辑、删除策略配置
- 表单参数根据后端返回的 schema 动态渲染

### 历史数据管理页

- 历史数据下载只选择交易对和周期，不再暴露时间范围输入
- 可选交易对来自后端配置 `app.backtest.supported_symbols`
- 可选周期来自后端配置 `app.backtest.supported_timeframes`，当前默认支持 `5m / 15m / 30m / 1h / 4h / 1d`
- 支持导入 zip 压缩包、导出选中文件为 zip 压缩包
- 文件列表展示文件名、交易对、周期、格式、大小、更新时间与数据覆盖范围
- 支持从历史数据列表一键跳转到策略回测页并带上历史文件
- 页面不再承载系统级目录提醒，相关路径与只读配置统一收口到系统设置页

### 策略回测页

- 回测主交互改为选择历史文件，而不是手工选择时间范围
- 选择文件后会自动展示交易对、周期和文件时间范围
- 点击“开始回测”不会直接创建任务，而是先弹出确认窗口，展示策略配置、策略类型、历史文件、交易对、周期、文件时间范围、初始资金与手续费率，确认后才真正提交
- 回测任务列表新增预计完成时间列，运行中或停止中的任务会随轮询刷新 ETA
- 回测任务列表与详情抽屉都支持停止任务；提交停止后任务会先进入“停止中”，最终收敛为“已停止”
- 详情抽屉会展示停止请求时间、当前进度、进度百分比，并兼容终态任务显示“用户已停止任务”提示
- 回测任务详情兼容新旧两类数据来源展示：
  - 新任务：显示历史文件来源信息
  - 旧任务：仍可展示旧的 symbol / timeframe / timerange 结构

### 系统设置页

- 集中展示历史数据目录、Freqtrade `user_data` 目录、系统日志目录等系统级路径
- 同时展示默认周期、历史数据格式、导出压缩格式、下载时间范围等只读配置
- 适合承载系统级信息，不再把这类说明散落在业务操作页顶部

### 系统日志页

- 支持按文件名与日志类型分页查询日志文件
- 支持在抽屉中查看指定日志文件最后若干行内容
- 日志详情在页面内滚动展示，不撑破主内容区

### 用户维护页

- 管理员可分页查询用户
- 支持新增用户、编辑用户、重置密码、切换状态

## 开发约束

- 页面交互调整后，应优先在浏览器里实际点一遍，而不只看构建是否通过。
- 需要新增固定选项的筛选器时，优先使用 `Select`、`DatePicker`、`RangePicker` 等组件，不要让用户承担无谓的自由输入成本。
- 无特殊要求时，前端界面文案、日期组件、日历面板和数字时间展示默认使用中文，不要退回英文 locale。
- 与后端联调时，优先保持接口值与后端真实枚举一致，例如交易日志中的 `strategy` 当前对应策略类型，而不是策略配置名称。
